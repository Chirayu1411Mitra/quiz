QuizLiveReal-Time Interactive Quiz PlatformFull Product Design &amp; Engineering SpecificationVersion1.0.0DateMay 2026TypeHackathon Project1. Product OverviewQuizLive is a real-time, browser-based interactive quiz platform inspired by Mentimeter and Slido. It enables a host (admin) to create and control quiz sessions while participants join via a short code and answer questions live from any device. Results are visualized in real time on both the admin dashboard and participant screens.1.1 Core Value PropositionNo app installation required — participants join via browser with a room codeAdmin has full session control — advance questions, lock/unlock answers, end sessionLive results update in real-time for all participants simultaneouslyLeaderboard updates after every question to drive engagementWorks on mobile, tablet, and desktop for all participants1.2 Target UsersUser TypeRolePrimary NeedAdmin / HostCreates &amp; controls quiz sessionFull control panel with live participant viewParticipantJoins &amp; answers questionsSimple, fast answer submission on any deviceViewer (optional)Watches on projector screenLive results &amp; leaderboard display2. Real-Time Socket.IO Event ProtocolArchitecture PatternAll real-time events flow through Socket.IO rooms keyed by roomCode.Admin joins room &apos;admin:{roomCode}&apos;. Participants join &apos;room:{roomCode}&apos;.Server relays broadcast events to all sockets in the relevant room.4.1 Admin-Emitted Events (Client → Server)EventPayloadDescriptionadmin:create_session{ quizId }Creates session, returns roomCodeadmin:start_session{ roomCode }Opens lobby → waiting for participantsadmin:next_question{ roomCode }Advances to next question, starts timeradmin:lock_answers{ roomCode }Prevents further answer submissionsadmin:show_results{ roomCode }Push current question results to alladmin:show_leaderboard{ roomCode }Broadcast top-N leaderboard to alladmin:end_session{ roomCode }Ends session, persists final scoresadmin:kick_participant{ roomCode, participantId }Removes a participant from session4.2 Participant-Emitted Events (Client → Server)EventPayloadDescriptionparticipant:join{ roomCode, nickname }Join session lobby, get assigned participantIdparticipant:submit_answer{ roomCode, participantId, questionId, answer }Submit answer for current questionparticipant:rejoin{ roomCode, participantId }Reconnect with existing identity after drop4.3 Server-Broadcast Events (Server → Clients)EventPayloadReceived Bysession:participant_joined{ participantId, nickname, count }Admin onlysession:participant_left{ participantId, nickname }Admin onlysession:question_start{ question, questionIdx, total, timeLimit }All participantssession:answer_update{ responseCounts, answeredCount, totalCount }Admin only (live chart)session:answer_submitted_ack{ pointsEarned, correct, rank }Submitting participant onlysession:answers_locked{ }All participantssession:results{ correctOption, distribution, stats }All participantssession:leaderboard{ topParticipants[], myRank }All participantssession:ended{ finalLeaderboard }All participantssession:kicked{ reason }Kicked participant onlysession:timer_sync{ secondsLeft }All participants (every second)5. Scoring AlgorithmSpeed-Based Scoring (Kahoot-style)Participants who answer correctly AND quickly earn more points.Maximum points = question&apos;s configured point value (default 1000).Points decay linearly based on time taken vs. total time limit.Wrong answers earn 0 points. Unanswered questions earn 0 points.5.1 Points Calculation// server/socket/handlers.tsfunction calculatePoints( isCorrect: boolean, timeElapsedMs: number, timeLimitMs: number, maxPoints: number): number { if (!isCorrect) return 0; // Time bonus: full points if answered instantly, // 50% of points minimum even if answered at the last second const timeRatio = timeElapsedMs / timeLimitMs; // 0.0 → 1.0 const multiplier = 1.0 - (timeRatio * 0.5); // 1.0 → 0.5 return Math.round(maxPoints * multiplier);}// Streak bonus (optional enhancement)function applyStreakBonus(basePoints: number, streak: number): number { if (streak &gt;= 3) return Math.round(basePoints * 1.1); // +10% if (streak &gt;= 5) return Math.round(basePoints * 1.25); // +25% return basePoints;}6. Key Implementation Details8.1 Room Code Generation// server/utils/roomCode.tsconst CHARS = &apos;ABCDEFGHJKLMNPQRSTUVWXYZ23456789&apos;; // no ambiguous charsexport function generateRoomCode(): string { return Array.from({ length: 6 }) .map(() =&gt; CHARS[Math.floor(Math.random() * CHARS.length)]) .join(&apos;&apos;);}// Ensure uniqueness by checking against active rooms in DB/memory8.2 Timer SynchronizationServer is the source of truth for timers to prevent cheating:On question_start: server records questionStartTime = Date.now()Server emits timer_sync every second with { secondsLeft } to all participantsClients display server-synced countdown, not their own local timerOn time expiry: server auto-calls lockAnswers() for the roomSubmissions with server timestamp &gt; (questionStartTime + timeLimit) are rejected8.3 Reconnection Handling// client/hooks/useSocket.tsconst socket = io(SERVER_URL, { reconnection: true, reconnectionDelay: 1000, reconnectionAttempts: 5});// On reconnect, re-emit participant:rejoin with stored participantIdsocket.on(&apos;reconnect&apos;, () =&gt; { const { roomCode, participantId } = getStoredSession(); if (roomCode &amp;&amp; participantId) { socket.emit(&apos;participant:rejoin&apos;, { roomCode, participantId }); }});8.4 Answer Submission IdempotencyEach participant can only submit once per question — enforced server-sideparticipant.hasAnswered flag in RoomState checked on every submit eventDuplicate submissions are silently ignored and ack&apos;d as already submitted8.5 Socket.IO Server Setup// server/index.tsimport { createServer } from &apos;http&apos;;import { Server } from &apos;socket.io&apos;;import express from &apos;express&apos;;const app = express();const httpServer = createServer(app);const io = new Server(httpServer, { cors: { origin: process.env.CLIENT_URL, methods: [&apos;GET&apos;, &apos;POST&apos;] }});io.on(&apos;connection&apos;, (socket) =&gt; { socket.on(&apos;admin:create_session&apos;, handleCreateSession(io, socket)); socket.on(&apos;admin:next_question&apos;, handleNextQuestion(io, socket)); socket.on(&apos;admin:lock_answers&apos;, handleLockAnswers(io, socket));
